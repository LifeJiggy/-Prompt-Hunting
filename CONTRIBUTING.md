## Contribution Process

### 1. Preparation
- Fork the repository
- Create a feature branch: `git checkout -b feature/your-feature-name`
- Ensure your changes align with the project's ethical guidelines

### 2. Making Changes
- **Prompt Format**: Follow the established prompt structure:
  - Clear AI role definition
  - Key capabilities and advanced techniques
  - Structured analysis process
  - Ethical guidelines
  - Output format specifications
- **Documentation**: Update README.md and relevant documentation
- **Testing**: Validate your changes don't break existing functionality

### 3. Quality Standards
- **Technical Accuracy**: Ensure all security concepts are technically correct
- **Ethical Compliance**: All contributions must promote responsible disclosure
- **Clarity**: Write clear, actionable content that benefits the community
- **Completeness**: Include remediation advice and impact assessment

### 4. Testing Your Contribution
- **Peer Review**: Have another security professional review your changes
- **Practical Testing**: Test prompts with real (authorized) targets if possible
- **Documentation Review**: Ensure all changes are properly documented

### 5. Submission
- Commit your changes: `git commit -m "Add: [brief description of changes]"`
- Push to your fork: `git push origin feature/your-feature-name`
- Create a Pull Request with:
  - Clear title and description
  - Reference to any related issues
  - Screenshots or examples if applicable
  - Impact assessment of your changes

## Content Guidelines

### Prompt Structure
Each prompt should include:
```markdown
You are an elite [DOMAIN] AI, specializing in [SPECIALIZATION].

Your mission is to [MISSION STATEMENT].

Key Capabilities:
- [CAPABILITY 1]
- [CAPABILITY 2]
- [CAPABILITY N]

Advanced Techniques:
- [TECHNIQUE 1]
- [TECHNIQUE 2]
- [TECHNIQUE N]

Analysis Process:
1. [STEP 1]
2. [STEP 2]
3. [STEP N]

Ethical Guidelines:
- [GUIDELINE 1]
- [GUIDELINE 2]
- [GUIDELINE N]

Output Format:
- [FORMAT SPECIFICATION]

Example Query: "[EXAMPLE QUERY]"
```

### Ethical Standards
- **No Malicious Content**: Never include actual exploit code or destructive payloads
- **Responsible Disclosure**: Always emphasize ethical testing and proper reporting
- **Scope Respect**: Include scope validation in all testing methodologies
- **Impact Awareness**: Require impact assessment before exploitation

### Technical Standards
- **Current Knowledge**: Use up-to-date security knowledge and best practices
- **Tool Agnostic**: Design prompts to work with various tools, not just specific ones
- **Practical Focus**: Include real-world testing scenarios and limitations
- **Remediation Priority**: Always include prevention and mitigation strategies

## Code of Conduct

### Expected Behavior
- Be respectful and inclusive
- Focus on constructive feedback
- Maintain professional discourse
- Prioritize security and ethics

### Unacceptable Behavior
- Sharing unauthorized vulnerability information
- Promoting illegal or unethical activities
- Personal attacks or harassment
- Spreading misinformation

## Getting Help

If you need help with your contribution:
- Open a GitHub issue with "help wanted" label
- Join our community discussions
- Review existing issues for similar questions

## Review Process

All contributions undergo review by maintainers:
- **Initial Review**: Technical accuracy and ethical compliance (1-2 days)
- **Community Review**: Open for community feedback (3-5 days)
- **Final Approval**: Integration and documentation updates (1-2 days)

We aim to provide feedback within 7 business days of submission.
